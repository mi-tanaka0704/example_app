# Dart：List / Set / Map の違い（チートシート）

## まとめ
- **List**：順番のある「配列」。`index` で取り出す（0番目…）。
- **Set**：重複しない「集合」。同じ値は1つだけ。
- **Map**：`key -> value` の「辞書」。キーで値を引く。

---

## よく使う観点で比較

## List
- **順序**：ある（保持される）
- **重複**：OK
- **取り出し**：`list[i]`（添字）
- **向いてる用途**
  - 画面に並べる（ListViewなど）
  - 「順番が意味を持つ」データ（履歴、ランキング、手順）
  - 同じ要素が複数回出ても自然なケース

### 例
```dart
final todos = ['掃除', '買い物', '勉強'];
print(todos[0]); // 掃除
```
## Set

- **順序**：重視しない（集合として扱う）  
  - ※Dartの `Set` は挿入順を保持する実装が多い  
  - ただし **順序が必要なら List が安全**
- **重複**：NG（自動でユニーク化）
- **主な操作**：`contains`（存在確認）
- **向いてる用途**
  - 重複を消したい
  - 「含まれてる？」を高速に判定したい
  - タグ・権限・選択済みIDなど、集合として扱うもの

### 例
```dart
final selectedIds = <int>{1, 3, 5};
selectedIds.add(3); // 追加されない（重複しない）
print(selectedIds.contains(5)); // true
```

## Map

- **順序**：キーの挿入順を保持（多くのケースで）
- **重複**：キーは重複不可（値は重複OK）
- **取り出し**：`map[key]`
- **向いてる用途**
  - IDや名前などのキーから即取り出したい（辞書）
  - 設定値・集計結果・キャッシュ
  - データ変換（例：`id -> item` の索引を作る）

### 例
```dart
final userById = <int, String>{1: 'Tanaka', 2: 'Sato'};
print(userById[2]); // Sato
```

## 判断フロー

1. **キーで値を引きたい** → **Map**
2. **重複をなくしたい** → **Set**
3. **順番が重要 / indexで触りたい / UIに並べたい** → **List**
4. 迷うなら：**存在確認が多いならSet**、それ以外は **List**

---

## 使用例

- **List**：順番・UI・index
- **Set**：重複なし・contains
- **Map**：keyで一発アクセス
